# Aero EBT Integrations

AeroEBT supports comprehensive integrations with authentication providers, databases, data exchange systems, and airline management systems. This documentation provides IT departments with detailed guides for configuring and managing these integrations.

## Integration Categories

### Authentication

AeroEBT supports multiple authentication methods including enterprise Single Sign-On (SSO) and OAuth providers.

**Single Sign-On (SSO):**
- Azure AD SSO
- Google Workspace SSO
- Okta SSO

**OAuth Providers:**
- Google OAuth
- GitHub OAuth
- Apple Sign-In

For detailed authentication setup guides, see [Authentication Integrations](./integrations/authentication/overview.md).

### Data Exchange

Integration methods for exchanging data with external systems:

- REST API integration
- Webhook integration
- File-based integration
- Message queue integration

For detailed data exchange documentation, see [Data Exchange Integrations](./integrations/data-exchange/overview.md).

### Databases

Integration with database systems for data synchronization, reporting, and analytics:

- PostgreSQL
- MySQL / MariaDB
- Microsoft SQL Server
- Oracle Database
- Snowflake
- Amazon Redshift
- Google BigQuery

For detailed database integration guides, see [Database Integrations](./integrations/databases/overview.md).

### Airline Systems

Integration with airline management systems for operational efficiency:

- Crew Management Systems (CMS)
- Flight Operations Systems (FOS)
- Learning Management Systems (LMS)
- Human Resources Systems (HRIS)
- Aircraft Maintenance Systems (AMS)
- Regulatory and Compliance Systems

For detailed airline systems integration guides, see [Airline Systems Integrations](./integrations/airline-systems/overview.md).

## Getting Started

1. **Identify Requirements**: Determine which integrations are needed for your organization
